## #2 觀念翻轉

```
接昨天。

大家用 ChatGPT 都問一個問題拿一個答案。

我覺得那樣浪費。

真實工作不是這樣。

「Q2 這個檔期要做嗎？」這個決策牽扯：
• 策略定調
• 數據驗證
• 廣告素材
• 社群節奏
• 視覺方向
• 工程落地
• 測試驗收

七個角度。一個 ChatGPT 對話你怎麼問得完？

所以我把它拆成 14 個「同事」。

每個角色有明確專業和性格。

像我家廣告投手開口就問「ROAS 目標多少」；
內容主編最討厭 AI 味，會把「在當今這個時代」整句刪掉；
資安工程師看到 .env 出現在程式碼裡會抓狂。

它們會互相交棒。

# AI團隊 # 一人公司

明天介紹全團隊。
```

## #4 實戰示範

```
一個真實情境。

我：「XX 客戶廣告 ROAS 這週掉了 0.5」

Ada（數據）：
「先別急著換素材，漏斗看過嗎？
CTR 沒掉，轉換率掉 30% — 問題不在素材，在落地頁。」

Ken（工程）：
「上週我換過 CTA 元件，讓我看看...
手機版按鈕跑版，看不到。」

Hunter（廣告）：
「Ken 修之前我先暫停這組廣告，不然預算白燒。」

Zoe（特助）：
「我記進待辦，明早 10 點追蹤。」

4 個角色接力，5 分鐘結案。

換作單一 ChatGPT 會怎麼回？

「請提供更多廣告資訊以便分析...」

然後你一個人硬想。

# AI代理 # 多角色協作

明天最後一篇，給你 repo。
```
